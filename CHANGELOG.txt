================================================================================
 iCommunityChat - Changelog
================================================================================

v0.2.0 - Release
--------------------------------------------------------------------------------

 [NEW] Sticky Community Chat Mode (/cc)
  - Type /cc to enter community chat mode — works like /g for guild chat
  - Messages sent directly in the chat edit box without opening the roster window
  - Chat tab renamed to the active community name while in mode
  - Custom edit box header shows community name in iCC color
  - Typed text displayed in iCC color
  - /cc toggles mode on/off instantly on space (like /g, /s)
  - /cc <message> sends a quick message without entering sticky mode
  - Typing /s, /g, /p, /w etc. automatically exits community chat mode

 [CHANGED] Chat Output Format
  - Messages now display as [Community] [Player]: message
  - Brackets around player name colored in iCC orange

 [CHANGED] System Messages
  - All iCC system messages now route to all enabled chat frames
  - Previously only displayed in General chat

 [FIX] PendingInvites case mismatch — user input vs AceComm sender name
  - All PendingInvites keys normalized to lowercase
 [FIX] Whisper fallback now also checks roster membership before sending
 [FIX] Fallback sync request auto-adds pending invitees if reply was lost

v0.1.4 - Release
--------------------------------------------------------------------------------

 [FIX] Invite sync — invitee now sees full roster after accepting
  - Inviter is added to local community immediately on accept
  - Fallback sync request sent after 3 seconds if auto-sync doesn't arrive
 [FIX] Invite whisper fallback timer increased from 5s to 15s
  - AceComm message round-trip could exceed 5s, causing false "download" whispers
 [FIX] Guild and rank data now included in roster sync responses
 [FIX] Custom rank names included in roster sync for new members

v0.1.3 - Release
--------------------------------------------------------------------------------

 [NEW] Per-Character SavedVariables
  - Settings and communities are now stored per character, not account-wide
  - Alts must be independently invited to communities

 [NEW] Invite Token Security
  - Invitations now include a random security token
  - Invite replies are validated against the token to prevent forged joins
  - Pending invites expire after 120 seconds

 [NEW] Community Settings
  - Community name can be renamed by the leader
  - Custom rank names for Leader, Officer, and Member roles
  - Rank names synced to all members via settings broadcast

 [NEW] Per-Community Chat Output
  - Chat output tabs can be configured per community in settings
  - Mute toggle to suppress all chat output for a specific community

 [NEW] Enter Key Chat Focus
  - Pressing Enter while iCC frame is open focuses the community chat input

 [CHANGED] Empty State
  - Chat and roster panels are hidden when no communities exist
  - Clean empty state with centered Create Community button

 [CHANGED] Invite Whisper
  - Updated fallback whisper message for players without iCC installed

 [FIX] Invite ack key mismatch — realm suffix caused whisper to always fire
 [FIX] Clickable community links no longer trigger errors in other addons
 [FIX] Missing locale entry for CommunityName
 [FIX] Replaced broken TBC icon paths in community icon picker

v0.1.0 - Initial Release
--------------------------------------------------------------------------------

 [NEW] Community Management
  - Create cross-guild communities with /icc create <name>
  - Join up to 5 communities simultaneously
  - Community tabs for switching between multiple communities
  - Leave community with /icc leave (confirmation prompt)
  - Leader can disband community

 [NEW] Roster Window
  - Guild-style roster window toggled with /icc or minimap button
  - Members displayed with online status dot, class-colored name, rank, and level
  - Sorted: online first, then by role (Leader > Officer > Member), then alphabetical
  - Right-click context menu: Whisper, Promote, Demote, Kick (permission-gated)
  - Column headers: Name, Rank, Level

 [NEW] Private Chat
  - Chat panel integrated into the roster window
  - Messages displayed with optional timestamps and class-colored sender names
  - All communication via AceComm WHISPER (full privacy, no public channels)
  - Message history stored locally per community (max 200 messages)
  - Sound notification option for incoming messages

 [NEW] Invitation System
  - Invite players with /icc invite <name> or the Invite button
  - Invitation popup with Accept/Decline buttons (60s timeout)
  - Full roster sync sent to new members on join
  - Roster changes broadcast to all online members in real-time

 [NEW] Role System
  - Three roles: Leader, Officer, Member
  - Leader: invite, kick, promote, demote, disband
  - Officer: invite, kick
  - Member: chat only
  - Leadership transfer via promote to Leader (self-demotes to Officer)

 [NEW] Online Detection
  - Heartbeat system: 90-second interval broadcast to all community members
  - Members marked offline after 270 seconds without heartbeat
  - Auto-adjusts heartbeat interval for large communities (>20 members)
  - Online status reset on addon load (re-detected via heartbeat)

 [NEW] Settings
  - Debug Mode toggle
  - Show Timestamps in Chat toggle
  - Play Sound on New Message toggle
  - Show Minimap Button toggle

 [NEW] Minimap Button (LibDataBroker)
  - Left-click: Toggle roster window
  - Right-click: Open settings
  - Tooltip: Community list with online/total member counts

 [NEW] Slash Commands
  - /icc — Toggle roster window
  - /icc create <name> — Create a new community
  - /icc invite <player> — Invite a player to active community
  - /icc leave — Leave the active community
  - /icc settings — Open settings panel
  - /icc help — Show command help

 [TECHNICAL]
  - AceComm WHISPER-only protocol with 7 prefixes (Chat, Heartbeat, Invite, InvReply, Roster, Sync, Version)
  - Roster versioning with incremental sync
  - Sender validation on all incoming messages
  - Combat lockout: UI hidden during combat
  - Template compatibility: InterfaceOptionsCheckButtonTemplate / UICheckButtonTemplate, BackdropTemplate
  - Multi-TOC support: Classic Era, TBC, Wrath, Cata, MoP, Retail
